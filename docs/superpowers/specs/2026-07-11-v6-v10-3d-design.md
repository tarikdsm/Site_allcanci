# Especificação — Versões V6–V10 com modelo 3D da máquina FILL

Data: 2026-07-11

## Objetivo

Cinco novas propostas de site (V6–V10) para a Allcanci Tecnologia, cada uma com direção de arte
própria (paletas e fontes novas, derivadas mas distintas do manual original) e com o modelo 3D
`maquina_fill_web.glb` integrado de uma forma diferente em cada versão.

## Ativos 3D disponíveis

- `public/models/maquina_fill_web.glb` — modelo otimizado (~2,6 MB, ≤70k tris, materiais PBR,
  tela azul emissiva, logotipo dourado). Servido como asset estático: URL em runtime =
  `${import.meta.env.BASE_URL}/models/maquina_fill_web.glb`.
- Renders PNG transparentes 1600×1600 em `src/assets/3d/` (usar via `astro:assets`):
  - `maquina-fill-render.png` — frontal (2000×2000, o principal; poster/OG)
  - `maquina-fill-tq-esq.png` / `maquina-fill-tq-dir.png` — três-quartos esq./dir.
  - `maquina-fill-perfil.png` — perfil 90°
  - `maquina-fill-costas.png` — traseira
  - `maquina-fill-topo.png` — vista elevada (mostra os 7 encaixes coloridos do topo)

A máquina: corpo preto fosco, encaixes superiores azul/preto/vermelho, tela azul-escura emissiva,
placa frontal preta com logotipo "FILL Ink Injector" dourado, botões pretos com símbolos brancos.

## Convenções compartilhadas (OBRIGATÓRIAS para todas as versões)

### Arquivos por versão

Cada versão N cria EXATAMENTE 3 arquivos e não toca em nenhum outro:

- `src/pages/vN/index.astro`
- `src/styles/vN.css` (importado no frontmatter da página)
- `src/scripts/vN.js` (incluído com `<script src="../../scripts/vN.js"></script>` no fim)

Usar `Layout.astro` com `bodyClass="vN"`. Todo CSS com prefixo de classe `.vN-…` e variáveis
declaradas em `body.vN { --… }` para não vazar entre versões.

### Seções e ids (verificados por scripts/check-dist.mjs)

`hero`, `produtos`, `como-funciona`, `simulador`, `prova-social`, `sustentabilidade`, `faq`,
`contato` — todos presentes como `id=` de `<section>`, nesta ordem, com `aria-labelledby`.

### Conteúdo

Somente de `src/data/*.ts` (`site.ts`, `produtos.ts`, `comodato.ts`, `faq.ts`, `provaSocial.ts`,
`sustentabilidade.ts`, `imagens.ts`). Não inventar números nem depoimentos.
`provaSocial.depoimentos` está vazio de propósito — renderizar condicionalmente como na V1.
Voz pt-BR consistente com V1–V5. Rodapé com contatos/redes + link `“Ver as 10 propostas de design”`
para `${base}/`.

### Simulador (idêntico em contrato à V1)

Markup com ids `sim-professores` (range 1–200), `sim-professores-num` (number), leituras
`sim-recargas`, `sim-custo-fill`, `sim-descartaveis`, `sim-plastico`, `sim-economia`;
estado inicial renderizado no servidor com `simular(20)`; no JS,
`import { bindSimulador } from '../scripts/simulador-bind.js'`… (nos scripts de página o caminho é
`./simulador-bind.js`). Exibir as PREMISSAS como na V1.

### Fotos JPEG existentes

Usar `astro:assets` `<Image>` com `widths`/`sizes` e `alt` reais (copiar os alts da V1 quando a
mesma foto for usada). **`imagensEscurasProibidas`** (`ink-injector-frontal.jpg`,
`master-color-trio.jpg`, `master-color-vermelho.jpg`, `moedas-credito.jpg`) têm xadrez falso
gravado: só sobre fundo claro (#f0f0f0 ou mais claro) — nas versões escuras, colocar em “palcos”
claros como a V1 faz. Os renders PNG de `src/assets/3d/` têm alfa REAL: funcionam em qualquer fundo.

Convenção de palco de foto: `max-width/max-height + width/height:auto` — NUNCA `height:100%`.

### 3D — regras de performance e acessibilidade

- O elemento 3D nunca é o LCP: usar poster/imagem (render PNG→webp via `getImage`) visível de cara.
- Inicializar o 3D só quando a seção se aproxima do viewport (`IntersectionObserver`) e pausar o
  loop quando fora de tela.
- `prefers-reduced-motion: reduce` ⇒ sem auto-rotate/animação de câmera/scroll-jacking; o modelo
  continua interativo por arrasto, ou vira imagem estática quando indicado na versão.
- model-viewer: importar `'@google/model-viewer'` no vN.js; no elemento usar `camera-controls`,
  `poster={posterWebp.src}`, `alt` descritivo, `shadow-intensity`, `exposure`,
  `interaction-prompt="none"` (fazer o hint visual próprio), `disable-tap`? não — padrão.
  CSS: `model-viewer { --poster-color: transparent; }`.
- three.js: `import * as THREE from 'three'` + loaders de `'three/addons/…'`;
  `renderer = new THREE.WebGLRenderer({ alpha:true, antialias:true })`,
  `renderer.outputColorSpace = THREE.SRGBColorSpace`,
  `renderer.toneMapping = THREE.ACESFilmicToneMapping`, DPR ≤ 2, luzes: HemisphereLight +
  DirectionalLight key + rim; enquadrar via `new THREE.Box3().setFromObject(gltf.scene)` (não
  chumbar coordenadas); tratar `resize`; `WebGLRenderer` criado uma única vez.
- URL do GLB: calcular no frontmatter e passar ao DOM por atributo
  (`src={glbUrl}` no model-viewer; `data-glb={glbUrl}` no canvas/container para three.js).
- Sem fetch de CDNs: tudo via npm (já instalado: `@google/model-viewer`, `three`, fontes abaixo).

### Fontes

Importar SOMENTE no frontmatter da própria página (o Layout já traz Poppins/Nunito — ignorar e
sobrescrever via CSS). Pacotes já instalados por versão (ver cada spec).

### Acessibilidade e QA

Skip-link, foco visível, contraste AA, `role="list"` em listas estilizadas, botões com rótulos,
`aria-hidden` em decorações. Não rodar build/dev/npm — a verificação é central.

## V6 — “Vitrine” (showroom obsidiana & dourado) — model-viewer + hotspots

**Conceito.** A máquina tratada como peça de museu/joalheria. Palco preto profundo, tipografia
serifada de alto contraste, filetes dourados. O visitante gira a peça livremente e toca em
hotspots que revelam a ficha técnica.

**Paleta** (`body.v6`): `--obsidiana:#0b0b0f; --carvao:#15151c; --grafite:#23232e;
--dourado:#e6c87c; --dourado-claro:#f4e3b2; --marfim:#f6f1e4; --eletrico:#2e62ff;`
vermelho/azul dos encaixes apenas em micro-detalhes dos hotspots.

**Fontes.** `@fontsource-variable/fraunces` (display; pesos 500–900, usar também itálico via
font-style com a mesma família se disponível — senão só regular) + `@fontsource-variable/inter`
(UI/corpo). Números tabulares no simulador (`font-variant-numeric: tabular-nums`).

**Hero.** Etiqueta de exposição “Peça 01/06 — FILL Ink Injector · Betim, Brasil”; título serif
gigante “A máquina que aposenta o pincel descartável”; model-viewer ocupando ~60vh+, auto-rotate
lento (desligado em reduced-motion), fundo com vinheta radial e “pedestal” elíptico de luz.
3 hotspots (botões slot `hotspot-*`): encaixes do topo (“7 encaixes · azul, preto e vermelho”),
tela (“Visor com contagem de 22s por recarga”), placa (“Patente nacional · única no mundo”).
Posicionar hotspots dinamicamente no `load`: usar `mv.getDimensions()` e
`mv.getBoundingBoxCenter()` → topo = centro + y·0,5; frente = centro + z·0,5 (com
`data-normal` adequado), via `mv.updateHotspot(...)`. Painel lateral “ficha técnica” abre ao
clicar (um por vez, fechável, acessível).

**Seções.** Produtos = “galeria”: plaquetas numeradas 01–06, hairlines douradas, fotos sobre palco
marfim. Como-funciona = linha do tempo vertical com numerais serifados. Simulador = “avaliação da
peça”: console escuro, dígitos dourados tabulares. Sustentabilidade sobre verde-escuro profundo
(#0e1512) com dourado. FAQ minimal com regra dourada. Contato = cartão obsidiana com CTA dourado
(texto preto sobre dourado).

**Assinatura.** Auto-rotate pausa na interação e retoma após 4s; hotspots pulsam sutilmente.

## V7 — “Linha de Produção” (planta industrial) — three.js scroll-choreography

**Conceito.** O site é uma prancheta de engenharia viva: papel técnico, grade azul de planta,
carimbos, cotas em monoespaçada — e a máquina 3D gira conforme o scroll, como numa bancada de
inspeção.

**Paleta** (`body.v7`): `--papel:#f3efe4; --papel-2:#eae4d3; --tinta:#14141a;
--planta:#1d3fbf; --laranja:#ff5a1f; --grade:rgba(29,63,191,.13);` verde de aprovação
`#1a7f4b` só em carimbos.

**Fontes.** `@fontsource-variable/archivo` (títulos, peso 700–900, uppercase, tracking apertado) +
`@fontsource/archivo-black` (números gigantes de estação) + `@fontsource/ibm-plex-mono`
(anotações, cotas, rótulos).

**3D.** Canvas three.js `position:fixed` cobrindo a viewport, atrás do conteúdo (z-index abaixo;
`pointer-events:none`). GLTF carregado 1×; câmera orbita entre waypoints (azimute/elevação/zoom)
interpolados pelo progresso de scroll com easing suave (lerp por frame, fator ~0.06):
hero=frontal → produtos=três-quartos esq. → como-funciona=perfil → simulador=topo (mostra os
encaixes) → prova-social=três-quartos dir. → sustentabilidade/faq=frontal afastada → contato=frente
próxima. HUD mono fixo no canto: `AZ 035.2° · EL 12.0° · ZOOM 1.20×` atualizado por frame
(aria-hidden). Reduced-motion: canvas removido; render `maquina-fill-tq-esq.png` estático no hero.
Mobile estreito (<720px): canvas vira palco só do hero (não fixed), sem coreografia.

**Layout.** Fundo com grade de planta (repeating-linear-gradient fino). Conteúdo em cartões de
“papel” semi-opacos (para a máquina aparecer nos vãos), alternando esquerda/direita. Cada seção é
uma “ESTAÇÃO”: cabeçalho mono `EST. 01 — RECEPÇÃO` + número Archivo Black gigante fantasma.
Cotas decorativas (setas/medidas CSS) em volta das fotos; carimbos rotacionados “PATENTEADO”,
“APROVADO P/ LICITAÇÃO”. Simulador = “ordem de serviço” com campos mono. FAQ = notas de margem.

**Assinatura.** A coreografia de câmera + HUD técnico.

## V8 — “Quiosque Pop” (neo-brutalista) — model-viewer caixa de brinquedo

**Conceito.** Feira de ciências pop: cores chapadas, bordas pretas grossas, sombras duras,
adesivos tortos, marquee. A máquina é o brinquedo dentro de uma caixa de papelão gráfica —
“GIRE A MÁQUINA”.

**Paleta** (`body.v8`): `--violeta:#6c4df6; --acido:#d9ff3f; --papel:#fff3df; --preto:#131313;
--rosa:#ff7ac3; --branco:#fff;` Bordas `3px solid var(--preto)`, sombras `8px 8px 0 var(--preto)`.

**Fontes.** `@fontsource-variable/bricolage-grotesque` (display, pesos altos, títulos empilhados
misturando texto sólido e vazado via -webkit-text-stroke) + `@fontsource-variable/space-grotesk`
(corpo/UI).

**Hero.** Fundo violeta com padrão de cruzinhas; título empilhado gigante (“CHEGA DE PINCEL NO
LIXO”); marquee horizontal (“RECARGA EM 22s ★ +500 ESCOLAS ★ 1 KM DE ESCRITA ★ …”, pausa em
hover/reduced-motion). model-viewer dentro de card papel com borda grossa, adesivo rotacionado
“GIRE-ME ⟳” e etiqueta “brinquedo favorito das escolas”. Botão “DAR UM GIRO” = incrementa
`camera-orbit` theta em +360° com transição (via JS, respeitando reduced-motion → giro instantâneo
de 90°). `disable-zoom` para não brigar com o scroll.

**Seções.** Produtos: cards em cores alternadas (violeta/ácido/rosa/papel) com fotos em palcos
brancos, “etiquetas de preço” penduradas com os destaques. Como-funciona: 6 passos como cartelas
de jogo numeradas. Simulador: painel “PLACAR” estilo fliperama, números blocados; slider gordo.
Prova social: selos como bottons/adesivos. Sustentabilidade: papel + verde? usar ácido sobre preto.
FAQ: barras coloridas empilhadas. Contato: cartaz final ácido com CTA preto.

**Assinatura.** Caixa de brinquedo + botão de giro + sistema de adesivos.

## V9 — “Levitação” (aurora glass, claro) — model-viewer + scroll-orbit + AR

**Conceito.** Leveza premium: fundo quase branco, blobs aurora desfocados, vidro fosco, a máquina
FLUTUANDO com sombra suave — e um botão de AR (“veja em tamanho real na sua mesa”).

**Paleta** (`body.v9`): `--fundo:#fafbff; --texto:#1c2233; --teal:#19c3b1; --peri:#7b8cff;
--blush:#ffb2c7; --vidro:rgba(255,255,255,.55);` gradientes aurora com blur(80px) em pseudo-
elementos fixos (aria-hidden), `backdrop-filter: blur(14px)` nos cartões (com fallback opaco).

**Fontes.** `@fontsource-variable/outfit` (display, 600–800) + `@fontsource-variable/dm-sans`
(corpo).

**Hero.** Headline “O quadro branco ficou leve.”; model-viewer flutuando (bob suave por CSS
animation no elemento; off em reduced-motion) sobre elipse de sombra; chips de vidro com os 4
stats. Atributos: `camera-controls`, `ar ar-modes="webxr scene-viewer"`,
`touch-action="pan-y"`. Botão próprio estilizado ativando `mv.activateAR()` (esconder se
`mv.canActivateAR === false` após load — mostrar então um chip “AR disponível no celular”).
Scroll-orbit: conforme a página rola pelo hero e seções seguintes, interpolar `camera-orbit`
(theta −30°→+30°, radius aproxima) via rAF + scroll progress; model-viewer suaviza sozinho.

**Seções.** Tudo em cartões de vidro sobre as auroras; produtos em grid 3×2 com fotos (fundo dos
cards branco translúcido — xadrez ok pois quase branco... usar palco branco sólido dentro do card
para as 4 fotos proibidas); como-funciona em stepper horizontal com linha gradiente;
simulador = painel de vidro com números com transição animada (contagem suave no update);
sustentabilidade com aurora verde-teal; FAQ clean; contato = cartão gradiente teal→peri com CTA
branco.

**Assinatura.** AR real no celular + flutuação + órbita por scroll.

## V10 — “FILL OS” (firmware/terminal) — three.js com modos de visualização

**Conceito.** O site é o sistema operacional da máquina: interface de firmware, monoespaçada,
scanlines, log de boot digitando, e um VISOR.3D com modos [SÓLIDO] [ARAME] [RAIO-X].

**Paleta** (`body.v10`): `--fundo:#060b1c; --painel:#0a1228; --painel-2:#0e1834;
--ciano:#43e8ff; --ambar:#ffc14d; --verde:#3cf08a; --vermelho:#ff4757; --texto:#c9d7ee;
--borda:rgba(67,232,255,.25);` Overlay de scanlines + vinheta CRT sutil (opacity ~0.06,
pointer-events none).

**Fontes.** `@fontsource/chakra-petch` (display 600/700; títulos) + `@fontsource/ibm-plex-mono`
(todo o resto — corpo, labels, botões).

**Chrome.** Barra superior fixa: `FILL OS v2.6 · ALLCANCI TECNOLOGIA` + LED pulsante + relógio
HH:MM:SS (JS). Nav como comandos: `[01] PRODUTOS  [02] PROCESSO  [03] SIM  …`.

**Hero.** Split: esquerda = log de boot digitado linha a linha
(`> montando reservatorios [AZUL] [PRETO] [VERMELHO] … OK`, `> bomba de tinta … OK`,
`> creditos disponiveis … 26/professor·ano`, `> sistema pronto. recarga em 22s`), com cursor
piscando; reduced-motion = tudo visível sem animação. Direita = painel “VISOR.3D” com canvas
three.js: modelo com rotação ociosa lenta + arrasto (OrbitControls, zoom limitado, pan off).
Botões de modo (radiogroup acessível): SÓLIDO = materiais originais; ARAME = clone com
`wireframe:true` ciano (`MeshBasicMaterial`); RAIO-X = `MeshNormalMaterial` ou básico ciano
translúcido aditivo. Readout mono sob o canvas: `tris: 68.4k · modo: SOLIDO · rot: 0.2 rpm`.

**Seções.** Cada seção = “módulo” com cabeçalho mono estilo caixa (`╔═ MÓDULO 02 · PRODUTOS ═╗`
desenhado com borders CSS, não caracteres reais no acessível). Produtos = inventário: linhas/cards
com specs mono e foto em palco claro (janela “scanner”). Como-funciona = pipeline com setas e
status OK verde. Simulador = prompt: cabeçalho `$ fill sim --professores 20`, leituras como
saída de log; slider estilizado como barra de progresso ciana. Prova-social = registros de
auditoria (selos com checksum decorativo). Sustentabilidade = painel verde com métricas.
FAQ = `man fill` (perguntas como entradas de manual). Contato = `$ fill contato --whatsapp` com
CTA âmbar.

**Assinatura.** Boot log + modos de render do visor.

## Hub e verificação

- `src/pages/index.astro`: passa a listar 10 propostas em dois grupos — “Série 1” (V1–V5) e
  “Série 3D” (V6–V10, badge “3D interativo”); hero atualizado (“10 propostas · 1 decisão”).
- `scripts/check-dist.mjs`: hub exige links v1…v10.
- QA central: `npm run build`, `node scripts/check-dist.mjs` (hub + v1…v10), `astro check`,
  screenshots headless de V6–V10 (desktop e mobile) para revisão visual.

## Critérios de aceitação

1. As 8 seções com ids corretos em cada versão nova; check-dist verde para as 6 páginas + hub.
2. Modelo 3D interativo funcional em todas (com fallback reduced-motion) e poster imediato.
3. Nenhuma foto de `imagensEscurasProibidas` sobre fundo escuro.
4. Paletas/fontes distintas por versão, sem vazamento de estilos entre versões.
5. Build Astro sem erros; `astro check` sem erros novos.
